再也不用去按角落里的方向键了

本文最后更新于：5 个月前

利用一些软件，或者直接修改系统的键位配置文件来实现将方向键移到和 Vim 一样的键位。

其实这篇早就想要写了，一直没有下手，以前只有在 macOS 改了，直到拖到在 Linux 和 Windows 上也实现了，那就填了这坑吧，存下档，同时与你们分享。

0x01 macOS 端

所需软件：传说中的 Karabiner-Elements ，开发者文档 奉上。

Karabiner 十分强大，但我其实只用了其中的一个功能，就是改键规则。

进入之后切换到Complex modifications下Rules点击Add rule，弹出页面最上面有Import more rules from the Internet，点击之后会进入 Karabiner 的官方规则仓库，可以在这里寻找你想要的规则，十分丰富。

我只用了上图中的两个，即实现了按Caps_lock + h/j/k/l为左/下/上/右，好用到爆，即使在我切换电脑的英文键位为 Programmer Dvorak 以后，操作键位也没有变，应该是绑定了系统 key code 的。

0x02 Linux 端

Linux 端改键有几种方法，不过我用了其中风险最大的一种，并且因此开不了机（只是我自己作死，改错了文件，然后就无法加载了），然后我就格盘换了 Windows 7 。

本节基于 Arch Linux 编写。

思路来自：Linux 永久更改键盘设定 by Allen Chang ，这里只做整理分享。

Linux 装有 Xorg 服务的都另采用 X keyboard extension (XKB) 来定义键盘设定，文件存储于usr/share/X11/xkb目录下。我们需要修改的就是各位用的键盘的配置文件，通常使用 us 键盘的用户修改system下 us 文件即可。

以下引用内容来自上方链接文章。

以下是我对/usr/share/X11/xkb/symbols/us的更改（需要 root 权限）：

default  partial alphanumeric_keys modifier_keys
 xkb_symbols "basic" {
 
     name[Group1]= "English (US)";
 
     key.type = "ONE_LEVEL";
     key <CAPS>  {  [ ISO_Level3_Shift ]   };
     modifier_map Mod5 { ISO_Level3_Shift };
 
     key <RCTL>  {  [ Caps_Lock ]   };
 
     key.type = "THREE_LEVEL";
     (...)
     key <AD08> {   [         i,       I,       Home ]   };
     key <AD09> {   [         o,       O,        End ]   };
     (...)
     key <AC03> {   [         d,       D,     Return ]   };
     key <AC04> {   [         f,       F,     Return ]   };
     key <AC05> {   [         g,       G             ]   };
     key <AC06> {   [         h,       H,       Left ]   };
     key <AC07> {   [         j,       J,       Down ]   };
     key <AC08> {   [         k,       K,         Up ]   };
     key <AC09> {   [         l,       L,      Right ]   };
     key <AC10> {   [ semicolon,   colon,  BackSpace ]   };
     (...)
     key <AB07> {   [         m,       M,      Prior ]   };
     key <AB08> {   [     comma,    less,       Next ]   };
     (...)
 };
(...)

其中：

  • key.type = “THREE_LEVEL”;定义了下面的按键都有三种模式（一般/按 shift/按 Mode_switch）。
  • ISO_Level3_Shift相当于先前提到的 Mode_switch，一种 modifier，是 level3（Shift 就是 ISO_Level2_Shift，也可以定义更高的 level）。
  • Modifiers 不会受到 Shfit 改变行为，如 ctrl/alt/enter/backspace 等等。
  • 在[]內的是一個 group，你也可以设定多个 group，然后设定按键在这些 group 之间切换。

改完之后执行sudo dpkg-reconfigure xkb-data清空 XKB 的 cache，再用setxkbmap us设定只用 US layout，切换输入法应该就会生效了。因为是直接更改 default 的设定，有问题可能会让你的键盘不能正常运作，改的時候千万要小心。若沒把握，可以先看 这篇 的教学，设定额外的键盘，不会影响到 US 的设定。

使用 XKB 的好处是可以永久改变键盘设定，不受切换输入法及休眠的影响，並且可以做复杂的改变（定义多重 Level 及 Group 等等）；缺点是规则繁多，要花时间理解，且不小心可能会让键盘不能正常运作。

具体实现步骤也许有些许偏差，主要还是这篇文章咕了太久了，系统都重装好几遍了……大家有什么问题可下方留言告诉我，如果看不见下方评论系统也可发邮件给我，欢迎大家。

0x03 Windows 端

使用强大的 AutoHotkey。

本节参考 让 CapsLock 键更实用，键盘映射改造。

以下是 AutoHotkey 的执行脚本，我参考上面老哥的帖子修改的，只保留了最为核心的功能：

  • Capslock + hjkl 实现上下左右。（你们使用其他布局的也可以很方便的修改键位代码）
  • 只按下 Capslock 键实现大小写切换。
#Persistent
SetCapsLockState, AlwaysOff

; Capslock + hjkl (left, down, up, right)

Capslock & h::Send {Blind}{Left DownTemp}
Capslock & h up::Send {Blind}{Left Up}

Capslock & j::Send {Blind}{Down DownTemp}
Capslock & j up::Send {Blind}{Down Up}

Capslock & k::Send {Blind}{Up DownTemp}
Capslock & k up::Send {Blind}{Up Up}

Capslock & l::Send {Blind}{Right DownTemp}
Capslock & l up::Send {Blind}{Right Up}

Capslock::
If GetKeyState("CapsLock", "T") = 1
    SetCapsLockState, AlwaysOff
Else 
    SetCapsLockState, AlwaysOn
Return

以上代码写为一个后缀为 .ahk 的脚本，即可被 AutoHotkey 软件识别。

自动化的结果是可以将上述文件开机自启，也可使用由 AutoHotkey 安装自带的一个转化工具将脚本转为可执行文件，然后添加到启动项。
